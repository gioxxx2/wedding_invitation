# 隐私和安全指南

## Token 保存位置

**GitHub Token 保存在哪里？**

✅ **保存在浏览器本地存储（localStorage）中**
- 只有您访问页面的浏览器才能访问
- 不会上传到任何服务器
- 不会出现在代码中
- 不会提交到 GitHub 仓库

**安全性：**
- ✅ Token 只存在于您的浏览器中
- ✅ 其他人无法看到您的 Token
- ✅ 即使项目是公开的，Token 也不会泄露

## 项目是否需要改成私有？

### 情况分析

**如果项目是公开的：**
- ✅ 代码是公开的（没问题）
- ✅ Token 不会泄露（保存在浏览器本地）
- ⚠️ **但是**：如果数据保存到 GitHub，`data/guests.json` 文件也会是**公开的**
- ⚠️ 这意味着所有来宾信息（姓名、电话等）都会公开可见

**如果项目是私有的：**
- ✅ 代码是私有的
- ✅ Token 不会泄露
- ✅ `data/guests.json` 文件也是私有的
- ✅ 来宾信息不会被公开

## 建议

### 方案一：项目改为私有（推荐）

**优点：**
- ✅ 保护来宾隐私
- ✅ 来宾信息不会泄露
- ✅ 更安全

**缺点：**
- ⚠️ GitHub Pages 免费版不支持私有仓库
- ⚠️ 需要升级到 GitHub Pro 或使用其他方案

**如何改为私有：**
1. 访问：https://github.com/gioxxx2/wedding_invitation/settings
2. 滚动到底部
3. 找到 "Danger Zone"
4. 点击 "Change visibility" → "Make private"

### 方案二：不保存到 GitHub（推荐，如果项目必须公开）

**优点：**
- ✅ 项目可以保持公开
- ✅ GitHub Pages 可以正常使用
- ✅ 数据保存在浏览器本地，不会泄露

**缺点：**
- ⚠️ 数据不会自动同步到 GitHub
- ⚠️ 需要手动导出备份

**如何操作：**
- 不设置 GitHub Token
- 数据只保存在浏览器本地
- 定期使用"导出数据"功能备份

### 方案三：使用 GitHub Gist（折中方案）

**优点：**
- ✅ 可以创建私有的 Gist
- ✅ 数据保存在私有 Gist 中
- ✅ 项目可以保持公开

**缺点：**
- ⚠️ 需要修改代码实现

## 当前实现

**当前代码：**
- Token 保存在 `localStorage` 中（浏览器本地）
- 如果设置了 Token，数据会保存到 `data/guests.json`
- 如果项目是公开的，`data/guests.json` 也会是公开的

## 隐私建议

### 如果项目必须公开：

1. **不要设置 GitHub Token**
   - 数据只保存在浏览器本地
   - 不会泄露到 GitHub

2. **或者使用私有 Gist**
   - 需要修改代码实现
   - 数据保存在私有 Gist 中

3. **定期导出备份**
   - 使用"导出数据"功能
   - 保存到本地或私有云盘

### 如果项目可以改为私有：

1. **改为私有仓库**
2. **设置 GitHub Token**
3. **数据自动保存到 GitHub（私有）**

## 总结

**Token 安全性：** ✅ 安全，保存在浏览器本地，不会泄露

**项目隐私：**
- 如果项目是**公开的**，且设置了 Token → ⚠️ 来宾信息会公开
- 如果项目是**私有的**，且设置了 Token → ✅ 来宾信息是私有的
- 如果不设置 Token → ✅ 数据只保存在本地，不会泄露

**推荐做法：**
- 如果项目必须公开 → 不设置 Token，只使用本地存储
- 如果项目可以私有 → 改为私有，然后设置 Token
